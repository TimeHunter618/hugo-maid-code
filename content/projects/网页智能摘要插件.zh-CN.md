> 一款基于大语言模型的 Chrome 浏览器侧边栏插件，帮助用户快速总结网页内容，提升信息获取效率

---

## 项目概述

### 产品定位

WebSum 是一款基于大语言模型的 Chrome 浏览器侧边栏插件，帮助用户快速总结当前网页内容，提升信息获取效率。类似 Sider 的使用体验，但更轻量、更专注于网页摘要核心场景。

### 目标用户

| 用户画像 | 使用场景 |
|----------|----------|
| 研究人员 | 快速了解论文、技术文章核心内容 |
| 学生 | 总结课程资料、学习笔记 |
| 职场人士 | 快速浏览新闻、行业报告 |
| 内容创作者 | 收集素材、提炼观点 |

### 核心价值

- **高效获取信息**：一键生成网页摘要，30秒读完长文
- **多维度理解**：支持要点提取、通俗解释、深度分析等多种模式
- **随时随地可用**：侧边栏常驻，不打断浏览流程
- **数据隐私可控**：支持多种 LLM 提供商，API Key 本地加密存储

---

## 功能特性

### 核心功能

| 功能 | 说明 |
|------|------|
| 网页内容摘要 | 一键生成结构化摘要，支持快速摘要、要点提取、通俗解释、深度分析四种模式 |
| 上下文问答 | 基于当前网页内容进行对话式问答，支持多轮追问 |
| 多角色讨论 | 选择多个 AI 角色参与圆桌讨论（费曼、芒格、马斯克等 18 个蒸馏角色） |
| 收藏管理 | 将网页及其摘要保存到本地，支持搜索和标签分类 |
| 多模型支持 | 支持 8+ LLM 提供商（OpenAI、DeepSeek、Qwen、Gemini 等） |

### 摘要模式说明

- **快速摘要**（默认）：生成 3-5 句话的精简摘要
- **要点提取**：以列表形式提取 5-10 个核心要点
- **通俗解释**：用简单语言解释专业内容
- **深度分析**：分析文章结构、核心论点、论据支撑

### 架构特点

| 模式 | 说明 |
|------|------|
| 前端为主架构 | 核心逻辑在 Chrome 扩展前端运行 |
| 后端可选 | 后端服务作为可选组件，提供更强大的能力 |
| 直接调用 LLM | 支持前端直接调用 LLM API，无需后端 |
| 降级方案 | 所有后端依赖功能都有本地降级方案 |

## 技术实现

### 技术栈

| 分类 | 技术 |
|------|------|
| 前端 | Chrome Extension Manifest V3 |
| 后端 | FastAPI + Python（可选） |
| LLM 适配器 | openai-python SDK |
| 存储 | chrome.storage.local |
| 加密 | AES-GCM 256位 |

### 核心技术点

#### 内容提取双策略机制

Content Script 可能因安全策略未注入，采用双策略方案：
1. **策略一**：`chrome.tabs.sendMessage` 消息通信
2. **策略二**：回退到 `chrome.scripting.executeScript` 直接执行

#### API Key 加密存储

| 项目 | 详情 |
|------|------|
| 算法 | AES-GCM 256位 |
| 密钥派生 | PBKDF2 + SHA-256，迭代 100,000 次 |
| 存储 | 加密后的 Base64 字符串 |

#### 蒸馏角色系统

18 个角色的思维框架内置在 `skills.js`，涵盖思维大师（费曼、芒格）、实战派（马斯克、塔勒布）、AI 专家、内容创作、学习辅助等多个分类。
